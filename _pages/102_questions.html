#QuestionAnswer
1 What is a disk in the context of Linux storage? A physical container for data
2 What is a partition? A logical subset of a disk, created by MBR or GPT partitioning schemes. Every disk must have at least one partition.
3 What are the two main partitioning schemes used to create partitions? MBR (Master Boot Record) and GPT (GUID Partition Table)
4 What does a filesystem describe? The way information is stored on disk — how directories are organized, relationships between them, and where data for each file is located
5 What is the limitation of standard partitions that LVM solves? Partitions cannot span multiple disks. LVM allows multiple partitions to be combined, even across disks, into a single logical volume.
6 What does mounting a filesystem mean? Attaching a filesystem to a specific point in the system's directory tree, called a mount point. Example: mount /dev/sda1 /mnt/mymount
7 Why keep /boot on a separate partition? To ensure the system can still boot even if the root filesystem crashes
8 Why keep /home on a separate partition? To make it easier to reinstall the OS without the risk of accidentally deleting user data
9 Why keep /var on a separate partition? To prevent misbehaved processes that fill up /var from causing a kernel panic or corrupting the root filesystem
10 What files does the /boot partition contain? Files used by the bootloader to load the operating system (kernel images, initrd, GRUB files)
11 Where are GRUB 2 bootloader files stored? /boot/grub
12 Why is the boot partition usually located at the start of the disk? Legacy IBM PC BIOS used CHS addressing with a maximum of 1024 cylinders, resulting in a 528 MB limit. Placing /boot before cylinder 1024 ensures the kernel can always be loaded.
13 The boot partition should end before _____ to ensure legacy BIOS systems can always load the kernel. Cylinder 1024 (528 MB)
14 What is a good recommended size for the /boot partition today? Around 300 MB
15 What is the EFI System Partition (ESP) used for? Used by UEFI-based machines to store boot loaders and kernel images for installed operating systems
16 What filesystem format does the ESP use? A FAT-based filesystem
17 What is the partition ID for the ESP on a GPT-partitioned disk? C12A7328-F81F-11D2-BA4B-00A0C93EC93B
18 What is the partition ID for the ESP on an MBR-partitioned disk? 0xEF
19 Where is the ESP mounted on a Linux system? /boot/efi
20 What does /var contain? "Variable data" — files the system must be able to write to during operation, including system logs (/var/log), temp files (/var/tmp), and cached application data (/var/cache)
21 What is the default data directory for the Apache web server? /var/www/html
22 What is the default database directory for MySQL? /var/lib/mysql
23 What is the swap partition used for? To swap memory pages from RAM to disk as needed when RAM is full
24 What utility must be run to set up a swap partition before it can be used? mkswap
25 What is a swap file and why is it useful? A file (instead of a dedicated partition) used as swap space. Useful for quickly increasing swap space when needed without repartitioning.
26 According to Red Hat's recommendation, what is the suggested swap size for a system with less than 2 GB of RAM? 2× RAM (or 3× RAM if hibernation is needed)
27 According to Red Hat's recommendation, what is the suggested swap size for a system with 2–8 GB of RAM? Equal to RAM (or 2× RAM with hibernation)
28 According to Red Hat's recommendation, what is the minimum swap size for a system with 8–64 GB of RAM? Minimum 4 GB (or 1.5× RAM with hibernation)
29 What is the partition ID for Linux swap space under MBR? 0x82
30 What is the partition ID for Linux filesystems under MBR? 0x83
31 What is the partition ID for Linux LVM under MBR? 0x8E
32 What is the basic unit in LVM? The Physical Volume (PV) — a block device such as a disk partition or RAID array
33 What is a Volume Group (VG) in LVM? A grouping of Physical Volumes that abstracts the underlying devices and appears as a single logical device with combined storage capacity
34 What are extents in LVM? Fixed-sized pieces that Volume Groups are subdivided into. Extents on a PV are Physical Extents (PE); those on a Logical Volume are Logical Extents (LE).
35 What is a Logical Volume (LV) in LVM? A subdivision of a Volume Group that functions like a partition but with more flexibility (can be resized by adding or removing extents)
36 How is the size of a Logical Volume calculated? Physical extent size (4 MB by default) × number of extents on the volume
37 Where does a Logical Volume appear as a device file? /dev/VGNAME/LVNAME
38 What command is used to format a Logical Volume with ext4? mkfs.ext4
39 What does GRUB stand for? Grand Unified Bootloader
40 What was the last release version of GRUB Legacy and when was it released? Version 0.97, released in 2005
41 On an MBR-partitioned disk, what are the two things stored in the Master Boot Record? A partition table (lists up to 4 partitions) and bootstrap code (tiny bootloader)
42 How large is the MBR? 512 bytes
43 What is the MBR boot flow on a BIOS system? BIOS → MBR (tiny code) → 32 KB gap ("core image") → GRUB configuration → OS
44 What are the three limitations of MBR? Only 4 primary partitions; maximum 2 TB disk size; very limited bootloader code size
45 What is the UEFI boot flow? UEFI Firmware → ESP (FAT32) → grubx64.efi → GRUB → OS
46 What is the GRUB 2 EFI binary filename for 64-bit systems? grubx64.efi
47 What file is the kernel configuration stored in (example version: 4.15.0-65-generic)? config-4.15.0-65-generic — stores kernel configuration parameters. Generated automatically; should not be manually modified.
48 What is System.map-VERSION in /boot? A lookup table matching symbol names (variables/functions) to their corresponding memory positions
49 What is vmlinuz-VERSION? The compressed Linux kernel image (vmlinuz = compressed; vmlinux = uncompressed)
50 What is initrd.img-VERSION? The initial RAM disk — a minimal root filesystem loaded into RAM at boot. Modern systems use initramfs (initial RAM filesystem).
51 What is the default GRUB 2 configuration file and should it be edited directly? /boot/grub/grub.cfg — it is auto-generated and should not be edited manually
52 What file should be edited to change GRUB 2 settings? /etc/default/grub
53 After editing /etc/default/grub, what command regenerates the GRUB 2 configuration? update-grub (equivalent to grub-mkconfig -o /boot/grub/grub.cfg)
54 What does GRUB_DEFAULT= control? The default menu entry to boot. Can be a number (0 = first entry), a menu entry name, or saved.
55 What does GRUB_TIMEOUT=0 do? Boots the default entry immediately without showing the menu
56 What does GRUB_TIMEOUT=-1 do? Makes the system wait indefinitely for the user to select an option
57 What does GRUB_ENABLE_CRYPTODISK=y do? Makes grub-mkconfig, update-grub and grub-install look for encrypted disks and add the necessary commands to access them during boot. This disables automatic booting.
58 Where are custom GRUB 2 menu entries added? /etc/grub.d/40_custom
59 In a GRUB 2 menu entry, what does set root=(hd0,1) mean? Sets the root to the first disk (hd0), first partition (1). In GRUB 2, disks are numbered from 0, partitions from 1.
60 What does the linux line in a GRUB 2 menu entry specify? The path to the kernel image, followed by kernel command-line parameters (e.g. root=, ro, quiet)
61 What does the initrd line in a GRUB 2 menu entry specify? The path to the initial RAM disk image
62 How do you find the UUID of a filesystem partition? ls -l /dev/disk/by-uuid/
63 How do you tell GRUB 2 to find the root by UUID instead of device name? search --set=root --fs-uuid <UUID>
64 How do you bring up the GRUB 2 menu if only a countdown is visible? Press Shift
65 How do you enter the GRUB 2 shell from the menu screen? Press C (or Ctrl+C from the edit window)
66 What GRUB 2 shell command lists all detected disks and partitions? ls
67 What is the sequence of GRUB 2 shell commands to manually boot a Linux system? set root=(hd0,msdos1) → linux /vmlinuz root=/dev/sda1 → initrd /initrd.img → boot
68 What command prompt indicates the GRUB 2 rescue shell? grub rescue>
69 What two GRUB 2 modules must be loaded in the rescue shell before booting? insmod normal and insmod linux
70 In the rescue shell, what command sets the path to the GRUB 2 files? set prefix=(hd0,msdos1)/boot/grub
71 What is the command to install GRUB 2 to a disk that has a dedicated boot partition mounted at /mnt/tmp? grub-install --boot-directory=/mnt/tmp /dev/sda
72 What is the command to install GRUB 2 when there is no dedicated boot partition (just /boot on root)? grub-install --boot-directory=/boot /dev/sda
73 In GRUB Legacy, where are menu entries and settings stored? /boot/grub/menu.lst
74 In GRUB Legacy, how are disks and partitions numbered? How does this differ from GRUB 2? In GRUB Legacy, both disks and partitions are numbered from 0. In GRUB 2, disks start at 0 but partitions start at 1.
75 In GRUB Legacy, what does root (hd0,0) refer to? The first partition (0) of the first disk (hd0) — equivalent to /dev/sda1
76 What are the three required commands in a GRUB Legacy menu entry? title (OS name on menu), root (device/partition to boot from), kernel (path to kernel image)
77 From the GRUB Legacy shell, what are the commands to install GRUB Legacy to the first disk? grub> root (hd0,0) then grub> setup (hd0)
78 What is chainloading in GRUB Legacy? Loading an unsupported OS (like Windows) by first loading GRUB, then handing off to that OS's own bootloader
79 What command in a GRUB Legacy chainload entry sets the active partition flag? makeactive
80 What does chainload +1 do in a GRUB Legacy entry? Loads the first sector of the boot partition, where bootloaders are typically located
81 What command finds the partition containing /boot/grub/stage1 from the GRUB Legacy shell? find /boot/grub/stage1
82 What command identifies the boot partition on /dev/sda using a standard Linux tool? fdisk -l /dev/sda — the boot partition is marked with an asterisk (*) in the output
83 What is the file extension for a static library? For a shared library? Static: .a — Shared: .so (shared object)
84 What are the key differences between static and shared libraries? Static libraries are linked at compile time into the executable (larger binary, no runtime dependency). Shared libraries are loaded at runtime by the dynamic linker (can be shared by multiple programs, smaller binaries).
85 What is the shared library naming convention (soname)? lib<NAME>.so.<VERSION> — example: libmysqlclient.so.18.0.0
86 What are the common shared library directories on a Linux system? /lib, /lib32, /lib64, /usr/lib, /usr/lib64, /usr/local/lib
87 What is the difference between /lib and /usr/lib? /lib: libraries required by essential system binaries (/bin, /sbin), must be available at boot. /usr/lib: libraries for non-essential or additional software.
88 What is the dynamic linker search order for shared libraries? 1. LD_LIBRARY_PATH environment variable → 2. Paths in /etc/ld.so.conf.d/ and /etc/ld.so.conf → 3. Default system directories (/lib*, /usr/lib*)
89 What does /etc/ld.so.conf typically contain? A single include line: include /etc/ld.so.conf.d/*.conf
90 What is ldconfig used for? Creating and updating the shared library cache (/etc/ld.so.cache). Must be run after adding or updating library configuration files.
91 What does ldconfig -v do? Displays library version numbers, each directory name, and the symbolic links created
92 What does ldconfig -p do? Prints the list of directories and libraries stored in the current cache (/etc/ld.so.cache)
93 What is LD_LIBRARY_PATH and how is it set? An environment variable containing a colon-separated list of directories where shared libraries are searched. Set with: export LD_LIBRARY_PATH=/usr/local/mylib
94 What command removes the LD_LIBRARY_PATH variable? unset LD_LIBRARY_PATH
95 What command shows the shared library dependencies of an executable? ldd /path/to/executable
96 What does the ldd -u option show? Unused direct dependencies — libraries that are linked but not actually needed by the program
97 If you add a new shared library at /opt/lib/mylib, what two steps are needed to register it system-wide? 1. Create a .conf file in /etc/ld.so.conf.d/ containing the path /opt/lib/mylib. 2. Run ldconfig to update the cache.
98 How is LD_LIBRARY_PATH analogous to PATH? LD_LIBRARY_PATH is to shared libraries what PATH is to executables

151 What is a hypervisor? A software platform that runs processes containing fully emulated computer systems (virtual machines), managing the host's physical hardware resources among guest VMs
152 What is the difference between a Type-1 and Type-2 hypervisor? Type-1 (bare-metal) runs directly on hardware without an underlying OS (e.g. Xen). Type-2 requires a host OS to run (e.g. VirtualBox).
153 What type of hypervisor is KVM? Both Type-1 and Type-2 — it is a Linux kernel module that integrates with a running Linux installation but can perform as a hypervisor independently
154 What is a fully virtualized guest? A guest that is completely unaware it is a virtual machine. Requires Intel VT-x or AMD-V CPU extensions on x86 hardware.
155 What is a paravirtualized guest? A guest that is aware it is a virtual machine and uses a modified kernel and special guest drivers to communicate more efficiently with the hypervisor — often faster than full virtualization
156 What CPU extensions are required for full virtualization on x86 hardware? Intel VT-x or AMD-V (AMD SVM)
157 What disk image format does KVM use by default? What type is it? qcow2 — QEMU Copy-on-Write format (thin-provisioned: disk file grows as data is written)
158 What is a raw disk image? A disk image with all space pre-allocated. A 10 GB raw image consumes 10 GB on the hypervisor immediately. Offers better write performance than COW images.
159 What is the D-Bus machine ID and where is it stored? A unique machine identification number generated at install time. Stored at /var/lib/dbus/machine-id, symbolically linked to /etc/machine-id.
160 Why must a new D-Bus machine ID be generated when cloning a VM? No two Linux systems on the same hypervisor should share a D-Bus machine ID, as it is used to direct system resources to the correct guest
161 What command verifies that a D-Bus machine ID exists on the system? dbus-uuidgen --ensure
162 What command displays the current D-Bus machine ID? dbus-uuidgen --get
163 What commands regenerate a new D-Bus machine ID on a cloned VM? sudo rm -f /etc/machine-id then sudo dbus-uuidgen --ensure=/etc/machine-id
164 What command generates an SSH key pair for accessing cloud VMs? ssh-keygen — private key stored in ~/.ssh/, public key copied to the remote host
165 What command copies your SSH public key to a remote cloud server? ssh-copy-id -i user@cloud_server
166 What are the required file permissions for SSH private and public keys? Private key: 0600 — Public key: 0644
167 What is cloud-init? A vendor-neutral utility for pre-configuring cloud VMs and containers on first boot. Uses YAML (cloud-config) files to set hostname, packages, SSH keys, users, locale, and more.
168 What YAML syntax does cloud-init use, and what must be on the first line? The syntax is called cloud-config. The first line must be #cloud-config (no space between # and cloud-config).
169 What Linux kernel mechanism do containers use to partition system resources? Control groups (cgroups) — used to limit memory, processor time, disk and network bandwidth for individual applications
170 What command can be used to verify if CPU virtualization extensions are enabled? grep --color -E "vmx|svm" /proc/cpuinfo — vmx = Intel VT-x, svm = AMD-V
